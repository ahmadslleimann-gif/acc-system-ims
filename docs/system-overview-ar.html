🏍️ نظام كانيجيا للدراجات النارية

نظام سحابي للمحاسبة والمخزون والمبيعات والمشتريات — مع صلاحيات أمنية على مستوى المؤسسات.

Django REST + React/TypeScript · ثنائي اللغة (عربي/إنجليزي) · محاسبة القيد المزدوج

المحتويات

نظرة عامة التقنيات المستخدمة البنية المعمارية وحدات النظام الأدوار والصلاحيات التسعير والأمان دورة المخزون محرّك المحاسبة التقارير التدقيق والسلامة التشغيل محليًا واجهة الـ API

١ نظرة عامة

نظام كانيجيا للدراجات النارية هو نظام تخطيط موارد (ERP) مخصّص لأعمال تجارة الدراجات النارية. يدير المنتجات (المخزون) والعملاء والموردين والمبيعات والمشتريات، ويحتفظ تلقائيًا بدفاتر محاسبية صحيحة بنظام القيد المزدوج خلف الكواليس — فيحصل صاحب العمل على تقارير مالية دقيقة دون أي عمل محاسبي يدوي.

أصناف ومخزون

كل منتج هو صنف مخزون له تكلفة بالمتوسط المرجّح وكمية محدّثة لحظيًا.

الشراء يزيد المخزون

ترحيل فاتورة شراء يزيد المخزون ودَين المورّد تلقائيًا.

البيع ينقص المخزون

ترحيل فاتورة بيع يخصم المخزون ويسجّل التكلفة والإيراد ويرفع دَين العميل.

تسعير للمدير فقط

المدير العام وحده يحدّد الأسعار، والموظفون يبيعون بالسعر الرسمي.

ثنائي اللغة

عربي (RTL) وإنجليزي (LTR) بالكامل مع زر تبديل واحد.

سجل تدقيق

كل تغيير يُسجَّل: مَن ومتى والقيمة قبل/بعد.

٦ التسعير والأمان المالي

ثلاثة مستويات أسعار (للمدير العام فقط)

  • مفرّق · جملة · جملة الجملة
  • بالإضافة إلى سعر التكلفة وأدنى سعر بيع وأقصى نسبة خصم — كلها للمدير فقط.

الخادم هو مصدر الحقيقة

عند ترحيل عملية بيع، يتجاهل الخادم أي سعر يُرسله المتصفح ويحمّل السعر الرسمي والضريبة من بطاقة المنتج. أي طلب مُتلاعب به مثل { product:1, quantity:5, unit_price:1 } يُستبدَل بالسعر الحقيقي.
  • موظفو المبيعات لا يستطيعون تعديل الأسعار أو تجاوزها أو اختراعها (السطور بسعر مخصّص للمدير فقط).
  • البيع تحت أدنى سعر مرفوض (يمكن للمدير العام التجاوز).
  • أسعار التكلفة والأرباح تُحذف من استجابات الـ API للأدوار غير المخوّلة.

قفل المستندات

  • الفواتير والدفعات والقيود بعد الترحيل غير قابلة للتعديل.
  • السجلات المالية لا تُحذف أبدًا: الحالات هي مسودة ← مُرحَّل ← مُلغى.
  • إلغاء فاتورة مُرحَّلة يعكس القيد ويعيد المخزون — مع أثر تدقيقي كامل.

٧ دورة المخزون

شراء ١٠ بسعر ٨٠← المخزون ١٠، تكلفة ٨٠← بيع ٣ بسعر ١٢٠← المخزون ٧ (خصم تلقائي ٣)← تكلفة ٢٤٠، إيراد ٣٦٠
  • طريقة التقييم: المتوسط المرجّح للتكلفة، يُعاد حسابه مع كل استلام.
  • الكميات تتغيّر فقط عبر الشراء والبيع والمرتجعات وتسويات المدير — وكل تغيير يُسجَّل في حركة مخزون (دفتر مخزون كامل).
  • البيع بأكثر من المتوفر مرفوض.
  • كل منتج يتتبّع الموردين الذين يبيعونه مع تكلفة كل مورّد (قابلة للطباعة).

٨ محرّك المحاسبة (القيد المزدوج)

كل عملية تُنتج قيدًا متوازنًا. أمثلة:

الحدثمديندائن

+ المخزون المُباعتكلفة البضاعة المباعةالمخزون

المحرّك يرفض أي قيد غير متوازن، وأي ترحيل في فترة مغلقة، وأي ترحيل لحساب غير قابل للترحيل — ويكتب كل شيء في معاملة واحدة ذرّية.

١٠ التدقيق والسلامة

  • سجل التدقيق يحفظ المستخدم والإجراء والكيان والقيمة قبل/بعد وعنوان IP والوقت — للإضافة فقط ولعرض المدير فقط.
  • أرصدة العملاء والموردين تُحسب دائمًا من الفواتير والدفعات والإشعارات — غير قابلة للتعديل المباشر.
  • أرصدة النقدية/البنك مشتقة من قيود الدفتر.
  • المبالغ تستخدم أرقامًا عشرية ثابتة (بلا أخطاء تقريب).

١١ التشغيل محليًا

الخادم (Backend)

cd backend
.venv\Scripts\activate
python manage.py migrate
python manage.py seed_coa
python manage.py seed_roles
python manage.py runserver        # http://127.0.0.1:8000

الواجهة (Frontend)

cd frontend
npm install
npm run dev                        # http://localhost:5173

تصفير كل البيانات (مع الإبقاء على المستخدمين والحسابات والإعدادات)

python manage.py reset_data --yes
بيانات الدخول الافتراضية: admin / Admin12345 — يُرجى تغييرها بعد أول دخول. لوحة إدارة Django على http://127.0.0.1:8000/admin/.
